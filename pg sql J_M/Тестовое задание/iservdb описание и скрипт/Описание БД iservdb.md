# Описание базы данных ISERVDB

## Структура таблиц

### 1. Таблица `SD_Subscrs` (Лицевые счета)

| Поле | Тип | Ограничение | Описание |
|------|-----|-------------|----------|
| `LINK` | `int` | `PRIMARY KEY`, `GENERATED BY DEFAULT AS IDENTITY` | Уникальный идентификатор |
| `C_Number` | `text` | `NOT NULL`, `UNIQUE` | Номер лс |
| `C_FirstName` | `text` | `NOT NULL` | Имя |
| `C_SecondName` | `text` | `NOT NULL` | Фамилия |
| `C_Address` | `text` | `NULL` | Адрес регистрации |
| `C_Doc_Serial` | `text` | `NULL` | Серия паспорта |
| `C_Doc_Number` | `text` | `NULL` | Номер паспорта |
| `D_BirthDate` | `date` | `NOT NULL` | Дата рождения |

### 2. Таблица `DD_Docs` (Документы)

| Поле | Тип | Ограничение | Описание |
|------|-----|-------------|----------|
| `LINK` | `int` | `PRIMARY KEY`, `GENERATED BY DEFAULT AS IDENTITY` | Уникальный идентификатор |
| `C_Number` | `text` | `NULL` | Номер документа |
| `F_Subscr` | `int` | `NOT NULL` | ЛС |
| `C_Doc_Type` | `text` | `NOT NULL` | Тип документа |
| `D_Date` | `date` | `NOT NULL DEFAULT CURRENT_DATE` | Дата документа |
| `F_Docs` | `int` | `NULL` | Ссылка на родительский документ (самоссылка) |
| `B_Done` | `boolean` | — | Признак исполненного (TRUE - да, FALSE - нет) |


### 3. Таблица `DD_Docs_Assignments` (Мероприятия)

| Поле | Тип | Ограничение | Описание |
|------|-----|-------------|----------|
| `LINK` | `int` | `PRIMARY KEY`, `GENERATED BY DEFAULT AS IDENTITY` | Уникальный идентификатор |
| `F_Docs` | `int` | `NOT NULL` | Документ |
| `C_Name` | `text` | `NULL` | Наименование |
| `D_NeedToDo_Date` | `date` | `NOT NULL` | Плановая дата исполнения |
| `D_Done_Date` | `date` | `NULL` | Фактическая дата исполнения |
| `B_Done` | `boolean` | — | Признак исполненного (TRUE - да, FALSE - нет) |


### 4. Таблица `FD_Bills` (Счета)

| Поле | Тип | Ограничение | Описание |
|------|-----|-------------|----------|
| `LINK` | `int` | `PRIMARY KEY`, `GENERATED BY DEFAULT AS IDENTITY` | Уникальный идентификатор |
| `C_Number` | `text` | `NULL` | Номер |
| `F_Subscr` | `int` | `NOT NULL` | ЛС |
| `C_Sale_Items` | `text` | `NOT NULL` | Услуга |
| `D_Date` | `date` | `NOT NULL DEFAULT CURRENT_DATE` | Дата |
| `N_Amount` | `numeric(19,4)` | `NOT NULL` | Сумма |
| `N_Amount_Rest` | `numeric(19,4)` | `NOT NULL` | Остаток |


### 5. Таблица `FD_Payments` (Платежи)

| Поле | Тип | Ограничение | Описание |
|------|-----|-------------|----------|
| `LINK` | `int` | `PRIMARY KEY`, `GENERATED BY DEFAULT AS IDENTITY` | Уникальный идентификатор |
| `C_Number` | `text` | `NULL` | Номер |
| `F_Subscr` | `int` | `NOT NULL` | ЛС |
| `D_Date` | `date` | `NOT NULL DEFAULT CURRENT_DATE` | Дата |
| `N_Amount` | `numeric(19,4)` | `NOT NULL` | Сумма |

### 6. Таблица `FD_Payment_Details` (Детализация платежей)

| Поле | Тип | Ограничение | Описание |
|------|-----|-------------|----------|
| `F_Payments` | `int` | `NOT NULL` | Ид платежа |
| `F_Bills` | `int` | `NOT NULL` | Ид счета |
| `C_Sale_Items` | `text` | `NOT NULL` | Услуга |
| `N_Amount` | `numeric(19,4)` | `NOT NULL` | Сумма |

### 7. Таблица `ED_Devices` (Приборы учёта)

| Поле | Тип | Ограничение | Описание |
|------|-----|-------------|----------|
| `LINK` | `int` | `PRIMARY KEY`, `GENERATED BY DEFAULT AS IDENTITY` | Уникальный идентификатор |
| `F_Subscr` | `int` | `NOT NULL` | ЛС |
| `C_Name` | `text` | `NOT NULL` | Наименование |
| `C_Serial_Number` | `text` | `NOT NULL` | Серийный номер |
| `C_Sale_Items` | `text` | `NOT NULL` | Услуга |
| `D_Setup_Date` | `date` | `NOT NULL` | Дата установки |
| `D_Replace_Date` | `date` | `NULL` | Дата снятия |

### 8. Таблица `ED_Meter_Readings` (Показания приборов учёта)

| Поле | Тип | Ограничение | Описание |
|------|-----|-------------|----------|
| `LINK` | `int` | `PRIMARY KEY`, `GENERATED BY DEFAULT AS IDENTITY` | Уникальный идентификатор |
| `F_Devices` | `int` | `NOT NULL` | ПУ |
| `D_Date` | `date` | `NOT NULL` | Дата показания |
| `N_Value` | `decimal(19,6)` | `NOT NULL` | Значение |
| `C_Status` | `text` | `NULL` | Статус: "Дост", "Недост" |

### 9. Таблица `ES_Tariff` (Тарифы)

| Поле | Тип | Ограничение | Описание |
|------|-----|-------------|----------|
| `LINK` | `int` | `PRIMARY KEY`, `GENERATED BY DEFAULT AS IDENTITY` | Уникальный идентификатор |
| `C_Name` | `text` | `NOT NULL` | ПУ |
| `N_Tariff` | `decimal(19,6)` | `NOT NULL` | Значение |
| `C_Sale_Items` | `text` | `NOT NULL` | Услуга |
| `D_Date_Begin` | `date` | `NOT NULL` | Дата начала действия |
| `D_Date_End` | `date` | `NULL` | Дата окончания действия |

## Диаграмма связей (ER-диаграмма)

```
SD_Subscrs (1) ──┐
                 ├─ (1) DD_Docs (1) ─── (1..*) DD_Docs_Assignments
                 ├─ (1) FD_Bills (1) ── (0..*) FD_Payment_Details (0..*) ── (1) FD_Payments
                 └─ (1) ED_Devices (1) ── (1..*) ED_Meter_Readings

ES_Tariff (независимая справочная таблица)
```
